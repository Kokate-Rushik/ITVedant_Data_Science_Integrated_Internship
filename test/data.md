This Exploratory Data Analysis (EDA) report details a week-long investigation into a phishing URL detection dataset containing 11,430 records and 89 features. The dataset is perfectly balanced, with a 50/50 split between legitimate and phishing URLs, allowing for model evaluation without specialized resampling techniques.The features are categorized into four logical groups: lexical URL structures, word/token features, HTML content, and external service/WHOIS data. Statistical analysis revealed that most lexical features are heavily right-skewed, while page_rank and domain_age follow more normal distributions. A critical finding was the presence of six zero-variance features, such as submit_email and sfh, which carry no discriminative power and must be dropped before modeling.The study identified page_rank as the strongest predictor of legitimacy, noting that phishing sites typically have low or zero ranks because they are newly created and unindexed. Other strong indicators of phishing include high ratios of digits in the URL, longer URL lengths, and an increased number of dots and hyphens. Conversely, legitimate sites are more likely to have older domains and include the domain name in the page title.For upcoming modeling, the report recommends using tree-based ensembles like Random Forest or XGBoost, as they handle the identified outliers and skewed distributions effectively. It also suggests applying log transformations to skewed features and addressing negative sentinel values in the domain age data.
